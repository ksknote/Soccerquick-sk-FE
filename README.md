### 프로젝트 주제

<hr>

- 목적: 다분화된 풋살 매칭 플랫폼에서 주관하는 경기들을 한 곳에 모아 위치, 시간, 가격 등을 비교할 수 있는 플랫폼 구축 `<br>`
- 목표:

  - 사용자 경험 향상을 위해 메인페이지를 단순한 구성
  - 소셜 로그인 기능 및 채팅 기능을 제공
  - 위치 기반 경기장 검색을 지도에 나타냄으로 편리성을 제공
  - 각 플랫폼 및 경기장에 관한 리뷰 기능
  - 크롤링을 통해 수집한 대량의 데이터를 가공하여 사용자에게 양질의 정보 제공

### 서비스 소개

<hr>

#### User

- 사용자는 회원가입, 로그인이 가능하다.
- 사용자는 경기장을 즐겨찾기 할 수 있다
- 사용자는 마이페이지에서 회원정보 수정, 자신이 작성한 글/댓글, 즐겨찾기 목록을 볼 수 있다.
- 사용자는 회원 탈퇴가 가능하다. (이름/메일/전화번호가 6개월간 탈퇴회원DB에 저장된다)

#### Playground

경기장 정보는 매일 자정, 혹은 12시간 or 6시간 주기로 각 사이트에서 크롤링하여 수집한다.
수집된 정보는 DB에 저장되고, 아래 요청에 따라 요청된다.

- 메인 페이지
  1. 메인페이지에서 검색어를 입력하여 경기장 정보를 조회할 수 있다.
  2. 검색어를 입력하지 않은 경우 모든 경기장을 pagenation하여 조회할 수 있다.
  3. 검색어를 입력한 경우, 입력한 검색어에 맞는 경기장을 pagenation하여 조회할 수 있다.
- 리스트 페이지
  1. 경기장 정보를 카테고리별로 filter하여 보여줄 수 있다.
  2. 경기장 정보를 조건별로 sorting하여 보여줄 수 있다.
  3. 경기장을 클릭하면 상세 정보 페이지로 넘어간다
  4. 경기장을 여러개 선택하여 조건을 비교할 수 있다.(모달 창 띄움)
- 상세 정보 페이지
  1. 상세 정보페이지에서는 크롤링해온 경기장 정보를 자세하게 나타내준다.
  2. 상세 페이지에서는 해당 정보를 제공한 사이트로 이동하는 링크를 제공한다.
- 지도 서비스 제공
  1. 위치를 입력해서 근처 경기장을 조회할 수 있음.
  2. 이름으로 검색해서 경기장을 조회할 수 있음.
  3. 도별 선택 => 시별 선택 => 구별 선택 이후 해당 구에 있는 경기장을 지도로 보여줌.(API있음)

#### Community Page(자유게시판/리뷰게시판)

- 기본적인 커뮤니티 기능(작성, 수정, 삭제, 댓글에 대해서도 동일하게)

#### Team Page

- 팀페이지에서는 팀 모집, 모집중인 팀에 신청이 가능하다.
- 한시적 팀 모집글 : 모집자/지역/경기날짜/랜덤매칭수락여부/모집상태/모집인원(포지션)/사용(예정)경기장(필수아님)/성별/모집멘트
- 영구적 팀 모집글 : 모집자/지역/활동시간/랜덤매칭수락여부/모집상태/모집인원(포지션)/주 사용(예정)경기장(필수아님)/성별/모집멘트
- 일부 폼은 드롭다운으로 입력 받음(지역,모집상태,모집인원,성별)
- 날짜 선택은 달력API 활용 필요함.
- 신청글 : 신청자/지역/포지션/성별/신청멘트
- 랜덤매칭이 아닌 모집글은 작성자가 신청자의 정보를 확인하고 수락/거절할 수 있음.
- ★랜덤매칭 시스템 제공 : 조건이 맞는 신청자를 자동으로 수락하게 설정한 팀은, 조건이 맞는 신청자가 랜덤으로 매칭을 신청하여 합류랄 수 있음. (팀형태(한시적/영구적)/기타 필수사항들 입력 후 매칭)
- (23.05.30. 03:24 추가) 현재는 팀 모집글 기능만 있으나, 팀을 구하는(인재풀 등록) 기능을 추가하는 게 좋겠음.

##### 서버

- (user) 회원가입, 로그인, 회원정보 수정 등 유저 정보 관련 CRUD
- (user) 관리자 계정은 회원의 특정 기능(리뷰작성/커뮤니티 사용)을 정지시킬 수 있음
- (user) 관리자 계정은 회원정보를 수정하거나 강제 탈퇴 시킬 수 있음(추후 일정기간 재가입 불가) **탈퇴 시 회원정보를 남길 지, 탈퇴한 회원정보를 별도 데이터로 관리할 지 정해야함**
- (user) 마스터 계정은 유저 관리 페이지에서 일반 유저를 관리자로 임명할 수 있음
- (review) 리뷰 작성 관련 CRUD (리뷰 대상/평점/리뷰 멘트/etc....)
- (community) 게시물 작성 관련 CRUD, 페이지네이션 제공(query)
- (community) 댓글 작성 관련 CRUD, 페이지네이션 제공(query)
- (community) 관리자는 공지사항(상단 고정 게시글)을 작성할 수 있음(글 속성으로 관리해야 함)
- (playground) 검색(지도) 시 해당 구장 위치 정보(구장 이름, 위도, 경도 등) 제공
- (playground) 필터링(위치, 날짜) 조건에 따른 구장의 모든 정보 제공
- (playground) 경기장 검색 관련 페이지네이션 제공
- (playground) 경기 상세페이지에서 예약하기 버튼 클릭 시 특정 URL(해당 플랫폼의 경기 상세 페이지) 제공
- (playground) 현재 위치를 입력하면 현재위치에서 가장 가까운 풋살장 순으로(Haversine 공식) 제공
- (playground) 회원은 자주 찾는 경기장을 즐겨찾기(찜하기) 할 수 있음

### 팀 컨벤션

##### [Commit 컨벤션](https://velog.io/@shin6403/Git-git-%EC%BB%A4%EB%B0%8B-%EC%BB%A8%EB%B2%A4%EC%85%98-%EC%84%A4%EC%A0%95%ED%95%98%EA%B8%B0)

`태그 : 제목`의 형태이며, `:뒤에만 space`가 있음에 유의

- feat : 새로운 기능 추가
- fix : 버그 수정
- docs : 문서 수정
- style : 코드 포맷팅, 세미콜론 누락, 코드 변경이 없는 경우
- refactor : 코드 리펙토링
- test : 테스트 코드, 리펙토링 테스트 코드 추가
- chore : 빌드 업무 수정, 패키지 매니저 수정

##### 코드 컨벤션

- 파일명, export 함수명: PascalCase
- 주석:
  - 함수의 기능과 특징 설명 / 함수 위
  - 함수 내 코드 설명 / 코드 오른쪽
